# 事前課題6 採点基準

## 課題内容
テーマは「デバッグ」。レジ＋ポイント加算などの処理を題材に、**誤りが起こりそうな箇所を見つけ、その原因（なぜ誤りが起こりやすいか）を考える**。求められる内容は、(1)処理の流れ（擬似コードや状態変化表）、(2)**誤りが起こりそうな箇所を3つ以上**列挙、(3)各箇所について**なぜ誤るかの理由**、(4)処理の順序の誤り（例：残金を減らす前に確認するか後か、ポイント加算と会計の順序）への着目。

## 採点の方針
採点AIは主観で判断せず、**画像から観察できる事実**にYes/Noで答える。その観察結果から機械的に1〜3点を算出する。観点ラベルが無くても、図や文章から該当内容が読み取れれば「あり（Yes）」とみなす。

## 観察項目（画像を見てYes/Noで答える）
- **箇所Q1**: 「誤りが起こりそうな箇所／バグ／間違いやすい点」が**1つ以上**書かれているか
- **3つQ2**: その誤りの箇所が**3つ以上**列挙されているか
- **理由Q3**: 各誤りについて**なぜ誤るのか（原因・理由）**の記述があるか
- **順序Q4**: **処理の順序**の誤り（順番を入れ替えると結果が変わる等）への言及があるか
- **処理Q5**: 処理の流れを示す**擬似コード・状態変化表・トレース表・フロー図**のいずれかがあるか
- **量Q6**: 用紙のおよそ**7割以上**が文字・図・表・矢印で埋まっているか

## 必須観察数の数え方
**必須Yes数** = 箇所Q1・3つQ2・理由Q3・順序Q4・処理Q5 のうち「Yes」の個数（最大5）。

## 点数の決め方（3点満点・この順で判定）
1. **量Q6 が No**（白紙に近い・量が極端に少ない）、または **箇所Q1 が No**（デバッグ観点がほぼ無い）、または **必須Yes数が2以下** → **1点で確定**
2. **必須Yes数が4以上** かつ **3つQ2 が Yes** かつ **理由Q3 が Yes** → **3点**
3. それ以外（必須Yes数3、または誤り箇所が3つ未満・理由が欠ける） → **2点**

## 各点数の意味
- **3点**: 誤りの箇所を3つ以上挙げ、各原因の説明・処理順序への着目・処理の流れの整理まで揃った模範的レベル
- **2点**: 一般的レベル。デバッグ観点はあるが、数や理由づけに不足
- **1点**: 量が少ない・誤り箇所がほとんど挙がっていない・手抜き

## 出力JSONテンプレート
以下のJSON形式で回答すること。**観察項目をすべて記録すること**。

```json
{
  "点数": "<1, 2, または 3 の整数>",
  "観察項目": {
    "箇所Q1": "<Yes または No>",
    "3つQ2": "<Yes または No>",
    "理由Q3": "<Yes または No>",
    "順序Q4": "<Yes または No>",
    "処理Q5": "<Yes または No>",
    "量Q6": "<Yes または No>"
  },
  "観点別評価": {
    "誤り箇所の指摘": "<あり または なし>",
    "誤り箇所が3つ以上": "<あり または なし>",
    "原因・理由の記述": "<あり または なし>",
    "処理順序への着目": "<あり または なし>",
    "処理の流れの整理": "<あり または なし>",
    "量的充実": "<7割以上 または 不足>",
    "必須Yes数": "<0〜5の整数>/5"
  },
  "評価理由": "<観察項目のYes/Noと、それに基づく点数の決定根拠を明示すること>",
  "フィードバック": "<学生への建設的なコメントを日本語で>"
}
```

注意: 「点数」は観察項目から機械的に決まる。観察項目のYes/Noを先に確定させてから点数を出すこと。
