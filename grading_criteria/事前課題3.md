# 事前課題3 採点基準

## 課題内容
テーマは「条件分岐」。価格120円の自動販売機を例に、条件によって処理が変わる仕組みを整理する。求められる内容は、(1)「制御文（条件分岐）とは何か」を自分の言葉で説明、(2)入力・状態・条件・処理・状態更新・出力の6観点で整理、(3)条件分岐（在庫なし→「売り切れ」／在庫あり＆120円未満→「金額不足」／在庫あり＆120円以上→商品を出す）、(4)境界値（120円ちょうど）の確認、(5)複数ケースのトレース。

## 採点の方針
採点AIは主観で判断せず、**画像から観察できる事実**にYes/Noで答える。その観察結果から機械的に1〜3点を算出する。観点ラベルが無くても、図や文章から該当内容が読み取れれば「あり（Yes）」とみなす。

## 観察項目（画像を見てYes/Noで答える）
- **説明Q1**: 「条件分岐（制御文）とは何か」を**自分の言葉で説明した文章**があるか
- **観点Q2**: 入力・状態・条件・処理・状態更新・出力の**6観点のうち4つ以上**が整理されているか
- **分岐Q3**: **3つの結果（「売り切れ」「金額不足」「商品を出す」）**に対応する条件分岐が書かれているか
- **フローQ4**: **YES/NOの分岐を持つフローチャート、または判定のひし形・分岐図**があるか
- **境界Q5**: 条件式に**「120」という数値**（120円以上／未満、≧120 など）、または120円ちょうどの事例が登場するか
- **トレースQ6**: **複数ケース（例：A〜D、金額不足・売り切れ・購入など2件以上）を比べるトレース表・事例**があるか
- **量Q7**: 用紙のおよそ**7割以上**が文字・図・表・矢印で埋まっているか

## 必須観察数の数え方
**必須Yes数** = 説明Q1・観点Q2・分岐Q3・フローQ4・境界Q5・トレースQ6 のうち「Yes」の個数（最大6）。

## 点数の決め方（3点満点・この順で判定）
1. **量Q7 が No**（白紙に近い・量が極端に少ない）、または **必須Yes数が2以下** → **1点で確定**
2. **必須Yes数が5以上** かつ **境界Q5・トレースQ6 が両方Yes** → **3点**
3. それ以外（必須Yes数3〜4、または境界・トレースのどちらかが欠ける） → **2点**

## 各点数の意味
- **3点**: 6観点・条件分岐・フローチャートに加え、境界値と複数ケースのトレースまで確認している模範的レベル
- **2点**: 一般的レベル。条件分岐は整理されているが、境界値やトレースに不足
- **1点**: 量が少ない・条件が整理されていない・手抜き

## 出力JSONテンプレート
以下のJSON形式で回答すること。**観察項目をすべて記録すること**。

```json
{
  "点数": "<1, 2, または 3 の整数>",
  "観察項目": {
    "説明Q1": "<Yes または No>",
    "観点Q2": "<Yes または No>",
    "分岐Q3": "<Yes または No>",
    "フローQ4": "<Yes または No>",
    "境界Q5": "<Yes または No>",
    "トレースQ6": "<Yes または No>",
    "量Q7": "<Yes または No>"
  },
  "観点別評価": {
    "制御文の説明": "<あり または なし>",
    "6観点の整理": "<あり または なし>",
    "3分岐の記述": "<あり または なし>",
    "フローチャート": "<あり または なし>",
    "境界値120の確認": "<あり または なし>",
    "複数ケースのトレース": "<あり または なし>",
    "量的充実": "<7割以上 または 不足>",
    "必須Yes数": "<0〜6の整数>/6"
  },
  "評価理由": "<観察項目のYes/Noと、それに基づく点数の決定根拠を明示すること>",
  "フィードバック": "<学生への建設的なコメントを日本語で>"
}
```

注意: 「点数」は観察項目から機械的に決まる。観察項目のYes/Noを先に確定させてから点数を出すこと。
