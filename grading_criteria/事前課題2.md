# 事前課題2 採点基準

## 課題内容
テーマは「変数（状態の理解）」。自動販売機を例に、**処理の途中で保存しておくべき「状態」**を考える。具体的には、(1)何を保存するか、(2)それにどんな名前をつけるか、(3)最初はどんな状態か（初期値）、(4)どのタイミングで変化するか、(5)なぜ保存が必要か、を整理する。**「ボタンを押す」などの動作ではなく、「現在の投入金額」「在庫数」などの状態**を挙げることが求められる。

## 採点の方針
採点AIは主観で「良い・悪い」を判断せず、**画像から観察できる事実**にYes/Noで答える。その観察結果から、機械的に1〜3点を算出する。観点ラベルが無くても、図や文章から該当する記述が読み取れれば「あり（Yes）」とみなす。

## 観察項目（画像を見てYes/Noで答える）
- **状態Q1**: 「投入金額」「在庫」「選択商品」「おつり」などの**状態（名詞）が2つ以上**挙がっているか
- **状態Q2**: 挙げた状態に**名前（変数名）**がつけられているか（日本語名・英語名どちらでもよい）
- **状態Q3**: 各状態の**初期値・最初の状態**（例：投入金額は最初0円、選択商品は未選択）の記述があるか
- **状態Q4**: 状態が**変化するタイミング**（例：お金を入れたとき金額が増える）の記述があるか
- **状態Q5**: その状態を**なぜ保存する必要があるか（理由・用途）**の記述があるか
- **動作混同Q6**: 挙げている項目が「ボタンを押す」「商品を出す」などの**動作（動詞）ばかりに偏っていないか**（状態＝名詞が中心ならYes、動作ばかりならNo）
- **量Q7**: 用紙のおよそ**7割以上**が文字・図・表・矢印で埋まっているか
- **質Q8**: **図・表・矢印・具体例（例：150円入れる→…）・疑問や考察**のいずれかが含まれているか

## 必須観察数の数え方
**必須Yes数** = 状態Q1〜Q5 と 動作混同Q6 のうち「Yes」の個数（最大6）。

## 点数の決め方（3点満点・この順で判定）
1. **状態Q1 が No**（状態がほとんど挙がっていない）、または **動作混同Q6 が No**（動作ばかり）、または **量Q7 が No**（白紙に近い・量が極端に少ない） → **1点で確定**
2. 必須Yes数が **2以下** → **1点で確定**
3. 必須Yes数が **5以上** かつ **質Q8 が Yes** → **3点**
4. それ以外（必須Yes数3〜4、または質Q8がNo） → **2点**

## 各点数の意味
- **3点**: 状態・名前・初期値・タイミング・理由まで整理され、図表や具体例で充実。他者に共有したいレベル
- **2点**: 芝浦工大生の一般的レベル。状態は挙がるが初期値・理由などに一部不足
- **1点**: 量が少ない・動作と状態を混同している・手抜き

## 出力JSONテンプレート
以下のJSON形式で回答すること。**観察項目をすべて記録すること**。

```json
{
  "点数": "<1, 2, または 3 の整数>",
  "観察項目": {
    "状態Q1": "<Yes または No>",
    "状態Q2": "<Yes または No>",
    "状態Q3": "<Yes または No>",
    "状態Q4": "<Yes または No>",
    "状態Q5": "<Yes または No>",
    "動作混同Q6": "<Yes または No>",
    "量Q7": "<Yes または No>",
    "質Q8": "<Yes または No>"
  },
  "観点別評価": {
    "状態の列挙": "<あり または なし>",
    "変数名": "<あり または なし>",
    "初期値": "<あり または なし>",
    "変化のタイミング": "<あり または なし>",
    "保存の理由": "<あり または なし>",
    "動作と状態の区別": "<できている または 混同>",
    "量的充実": "<7割以上 または 不足>",
    "図表・具体例・考察": "<あり または なし>",
    "必須Yes数": "<0〜6の整数>/6"
  },
  "評価理由": "<観察項目のYes/Noと、それに基づく点数の決定根拠を明示すること>",
  "フィードバック": "<学生への建設的なコメントを日本語で>"
}
```

注意: 「点数」は観察項目から機械的に決まる。観察項目のYes/Noを先に確定させてから点数を出すこと。
