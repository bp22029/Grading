# 事前課題5 採点基準

## 課題内容
テーマは「組み合わせ（変数・条件・繰り返し）」。複数の商品を順番に確認し、買えるものだけを購入していく処理を整理する。求められる内容は、(1)**必要な変数の列挙と初期値**（残金・合計金額・購入数・商品番号など）、(2)**擬似コード**（「もし〜なら」「〜の間くり返す」を含む手順記述）、(3)**状態変化表**（具体例を1つ決め、各ステップで変数の値がどう変化するか）、(4)繰り返しの継続/終了条件。

## 採点の方針
採点AIは主観で判断せず、**画像から観察できる事実**にYes/Noで答える。その観察結果から機械的に1〜3点を算出する。観点ラベルが無くても、図や文章から該当内容が読み取れれば「あり（Yes）」とみなす。

## 観察項目（画像を見てYes/Noで答える）
- **変数Q1**: **必要な変数が複数列挙**されているか（残金・合計・購入数・商品番号など、2つ以上）
- **初期値Q2**: 変数の**初期値**の記述があるか
- **擬似Q3**: **擬似コード**（if／もし〜なら、繰り返し／〜の間 を含む手順記述）があるか
- **状態表Q4**: **状態変化表**（行＝ステップ／商品、列＝各変数の値）が**3行以上**あり、値の遷移が追えるか
- **終了Q5**: 繰り返しの**継続条件または終了条件**が書かれているか
- **量Q6**: 用紙のおよそ**7割以上**が文字・図・表・矢印で埋まっているか

## 必須観察数の数え方
**必須Yes数** = 変数Q1・初期値Q2・擬似Q3・状態表Q4・終了Q5 のうち「Yes」の個数（最大5）。

## 点数の決め方（3点満点・この順で判定）
1. **量Q6 が No**（白紙に近い・量が極端に少ない）、または **必須Yes数が2以下** → **1点で確定**
2. **必須Yes数が4以上** かつ **状態表Q4 が Yes** かつ **擬似Q3 が Yes** → **3点**
3. それ以外（必須Yes数3、または状態変化表・擬似コードのどちらかが欠ける） → **2点**

## 各点数の意味
- **3点**: 変数・初期値・擬似コードに加え、複数ステップの状態変化表まで揃い、途中の状態変化が追える模範的レベル
- **2点**: 一般的レベル。要素はあるが状態変化表や擬似コードに不足
- **1点**: 量が少ない・整理が不十分・手抜き

## 出力JSONテンプレート
以下のJSON形式で回答すること。**観察項目をすべて記録すること**。

```json
{
  "点数": "<1, 2, または 3 の整数>",
  "観察項目": {
    "変数Q1": "<Yes または No>",
    "初期値Q2": "<Yes または No>",
    "擬似Q3": "<Yes または No>",
    "状態表Q4": "<Yes または No>",
    "終了Q5": "<Yes または No>",
    "量Q6": "<Yes または No>"
  },
  "観点別評価": {
    "変数の列挙": "<あり または なし>",
    "初期値": "<あり または なし>",
    "擬似コード": "<あり または なし>",
    "状態変化表": "<あり または なし>",
    "継続・終了条件": "<あり または なし>",
    "量的充実": "<7割以上 または 不足>",
    "必須Yes数": "<0〜5の整数>/5"
  },
  "評価理由": "<観察項目のYes/Noと、それに基づく点数の決定根拠を明示すること>",
  "フィードバック": "<学生への建設的なコメントを日本語で>"
}
```

注意: 「点数」は観察項目から機械的に決まる。観察項目のYes/Noを先に確定させてから点数を出すこと。
