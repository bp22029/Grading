# 事前課題4 採点基準

## 課題内容
テーマは「繰り返し」。価格120円の自動販売機（500円投入・在庫3個）で、購入処理が**買える間くり返される**仕組みを整理する。求められる内容は、(1)「繰り返し処理とは何か」を自分の言葉で説明、(2)入力・状態・条件・処理・状態更新・出力の整理、(3)**何を繰り返すか・どの条件が成り立つ間くり返すか（継続条件）・何が起きたら終わるか（終了条件）**、(4)**繰り返すたびに投入金額と在庫がどう変化するかのトレース表**。

## 採点の方針
採点AIは主観で判断せず、**画像から観察できる事実**にYes/Noで答える。その観察結果から機械的に1〜3点を算出する。観点ラベルが無くても、図や文章から該当内容が読み取れれば「あり（Yes）」とみなす。

## 観察項目（画像を見てYes/Noで答える）
- **説明Q1**: 「繰り返し処理とは何か」を**自分の言葉で説明した文章**があるか
- **観点Q2**: 入力・状態・条件・処理・状態更新・出力の**6観点のうち4つ以上**が整理されているか
- **対象Q3**: **何を繰り返すのか**（購入処理を繰り返す等）が具体的に書かれているか
- **終了Q4**: **継続条件または終了条件**（在庫が0になったら終わる／残金が足りなくなったら終わる等）が書かれているか
- **トレースQ5**: **繰り返しごとに投入金額・在庫が変化する様子を追ったトレース表**（2行以上、複数回ぶんの値）があるか
- **量Q6**: 用紙のおよそ**7割以上**が文字・図・表・矢印で埋まっているか

## 必須観察数の数え方
**必須Yes数** = 説明Q1・観点Q2・対象Q3・終了Q4・トレースQ5 のうち「Yes」の個数（最大5）。

## 点数の決め方（3点満点・この順で判定）
1. **量Q6 が No**（白紙に近い・量が極端に少ない）、または **必須Yes数が2以下** → **1点で確定**
2. **必須Yes数が4以上** かつ **トレースQ5 が Yes** かつ **終了Q4 が Yes** → **3点**
3. それ以外（必須Yes数3、またはトレース・終了条件のどちらかが欠ける） → **2点**

## 各点数の意味
- **3点**: 6観点・継続/終了条件に加え、各回の投入金額・在庫の変化を追うトレース表まで揃った模範的レベル
- **2点**: 一般的レベル。繰り返しの要素はあるがトレースや終了条件に不足
- **1点**: 量が少ない・繰り返しの整理が不十分・手抜き

## 出力JSONテンプレート
以下のJSON形式で回答すること。**観察項目をすべて記録すること**。

```json
{
  "点数": "<1, 2, または 3 の整数>",
  "観察項目": {
    "説明Q1": "<Yes または No>",
    "観点Q2": "<Yes または No>",
    "対象Q3": "<Yes または No>",
    "終了Q4": "<Yes または No>",
    "トレースQ5": "<Yes または No>",
    "量Q6": "<Yes または No>"
  },
  "観点別評価": {
    "繰り返しの説明": "<あり または なし>",
    "6観点の整理": "<あり または なし>",
    "繰り返す対象": "<あり または なし>",
    "継続・終了条件": "<あり または なし>",
    "状態変化のトレース表": "<あり または なし>",
    "量的充実": "<7割以上 または 不足>",
    "必須Yes数": "<0〜5の整数>/5"
  },
  "評価理由": "<観察項目のYes/Noと、それに基づく点数の決定根拠を明示すること>",
  "フィードバック": "<学生への建設的なコメントを日本語で>"
}
```

注意: 「点数」は観察項目から機械的に決まる。観察項目のYes/Noを先に確定させてから点数を出すこと。
