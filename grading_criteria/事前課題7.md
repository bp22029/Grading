# 事前課題7 採点基準

## 課題内容
テーマは「総合問題」。**エレベーター**と**レジ**の**2つの題材**を、1枚のA3用紙に整理する。各題材について、(1)仕様の理解、(2)必要な変数の一覧（表形式）、(3)条件分岐、(4)繰り返し、(5)入れ子構造、(6)処理の流れ（フローチャートまたは擬似コード）、(7)状態変化表、(8)デバッグ観点（誤りが起こりそうな箇所3つ以上）を整理する。

## 採点の方針
採点AIは主観で判断せず、**画像から観察できる事実**にYes/Noで答える。その観察結果から機械的に1〜3点を算出する。観点ラベルが無くても、図や文章から該当内容が読み取れれば「あり（Yes）」とみなす。

## 観察項目（画像を見てYes/Noで答える）
- **2題材Q1**: **エレベーターとレジの2つの題材**について、それぞれ記述があるか（1題材しかない場合はNo）
- **変数表Q2**: いずれかの題材で**変数一覧（変数名・役割などの表）**があるか
- **条件Q3**: 条件分岐（もし〜なら）の記述があるか
- **繰り返しQ4**: 繰り返し処理の記述があるか
- **流れQ5**: **フローチャートまたは擬似コード**（処理の流れ図・手順記述）があるか
- **状態表Q6**: **状態変化表**（変数の値の遷移を追う表）があるか
- **デバッグQ7**: **デバッグ観点（誤りが起こりそうな箇所）**の記述があるか
- **量Q8**: 用紙のおよそ**7割以上**が文字・図・表・矢印で埋まっているか

## 必須観察数の数え方
**必須Yes数** = 変数表Q2・条件Q3・繰り返しQ4・流れQ5・状態表Q6・デバッグQ7 のうち「Yes」の個数（最大6）。

## 点数の決め方（3点満点・この順で判定）
1. **2題材Q1 が No**（題材が1つしかない）、または **量Q8 が No**（白紙に近い・量が極端に少ない）、または **必須Yes数が2以下** → **1点で確定**
2. **2題材Q1 が Yes** かつ **必須Yes数が5以上** → **3点**
3. それ以外（2題材はあるが必須Yes数3〜4） → **2点**

## 各点数の意味
- **3点**: 2題材とも、変数表・条件・繰り返し・処理の流れ・状態変化表・デバッグ観点まで多面的に整理した模範的レベル
- **2点**: 一般的レベル。2題材はあるが、一部の観点が欠ける
- **1点**: 題材が1つしかない・量が少ない・整理が不十分

```json
    "条件Q3": "<Yes または No>",
    "繰り返しQ4": "<Yes または No>",

    "状態表Q6": "<Yes または No>",
    "デバッグQ7": "<Yes または No>",
    "量Q8": "<Yes または No>"
  },
  "観点別評価": {
    "2題材の取組": "<あり または なし>",
    "変数一覧表": "<あり または なし>",
    "条件分岐": "<あり または なし>",
    "繰り返し": "<あり または なし>",

}
```

注意: 「点数」は観察項目から機械的に決まる。観察項目のYes/Noを先に確定させてから点数を出すこと。
